# 我把电脑上的 AI 编程助手，搬进了手机

> multi-cc-im —— 从手机远程驾驭跑在终端里的 Claude Code / Codex
>
> 长图文：每个「第 N 屏」是截图的天然切割点（5 屏 = 5 张图，想压成 4 张就把第 5 屏并进第 4 屏）。`> 配图` 标注建议在那一屏嵌一张产品截图，让长图既有文字又有真实界面。

## 第 1 屏 · 为什么

人在地铁上，笔记本扔在家里，一个 Claude Code agent 正跑到一半的重构。

以前这种时候我是抓瞎的：它干完了我不知道；它中途要问我"用方案 A 还是 B"，我也收不到；只能等回到工位。在公司大屏上跟 AI 结对编程很爽，可人一离开工位，就跟正在干活的 agent 彻底失联。

于是我写了 multi-cc-im：把跑在终端标签里的多个 AI 编程 agent，桥接到飞书。手机和电脑两头，都能驱动同一批 agent。

## 第 2 屏 · 回复，一张卡片就读完

> 配图 S1：飞书手机端，一条 cc 回复以卡片出现，markdown 原生渲染（标题 + 表格 + 代码块）。

agent 干完一轮，完整回复以一张卡片推到手机上 —— markdown 原生渲染，标题、表格、代码块、列表都在，不是被切成十几条乱序的纯文本碎片。

一条回复 = 一张卡 = 一条消息。在手机上读一份真实的 diff，第一次不觉得是受刑。

## 第 3 屏 · 派活像发消息，决策回个数字

> 配图 S2 / S3：手机发 `#frontend ...` 派活 + AskUserQuestion 选择题卡片。

给 agent 派活，就像发微信：

- 直接发文字 → AI 自动分诊到该处理的那个 agent
- `#frontend 把登录 bug 修了` → 精确发给名叫 frontend 的标签
- `#all 跑一下测试` → 广播给所有 agent

更妙的是反过来 —— agent 需要我拍板时：遇到方案分叉，它把"A / B / C 选哪个"以编号选项推到手机，我回个数字就定；要动 `.git` / `.env` 这类敏感文件，先在手机上问我"同意一次 / 始终允许 / 拒绝"。锁屏上就能批，不用 SSH 回去手忙脚乱。

## 第 4 屏 · 它最特别的地方：从不套壳

我最自豪的一点：它从不"套壳"你的 agent。

市面上很多 AI 工具是包一层自己的界面、接管你的输入输出。multi-cc-im 反着来 —— 你的 Claude Code 继续以原生的样子跑在终端标签里。这个 bridge 不启动它、不接管它的输入输出、不假装一层界面。

怎么做到的？出站（agent → 手机）纯靠 Claude Code 自己暴露的 hook；入站（手机 → agent）纯靠终端命令行把文字打进标签。你随时能走到电脑前，直接接手那个标签跟 agent 面对面 —— bridge 从不挡在中间。

零侵入、零套壳、零绑定。

## 第 5 屏 · 还有 + 怎么用

> 配图 S4：本地监控面板 `127.0.0.1:40719` 的 cost 标签（token + USD）。

其它顺手的：

- 本地一个监控面板，实时看哪些 agent 在跑、各烧了多少 token / 多少钱
- Claude Code 和 OpenAI Codex 能在同一个终端共存，都从一个手机寻址
- 所有数据只落本机，凭据加密存储，不上传任何服务器

它是我的个人项目，MIT 开源，我自己天天在用。

需要：一个飞书机器人 + WezTerm 或 iTerm2；启动向导会带你走完配置。

```bash
pnpm i -g multi-cc-im
```
